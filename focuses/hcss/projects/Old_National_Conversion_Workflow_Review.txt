## PROPOSED WORKFLOW (v1.0 – Pandoc-Compatible Slide Authoring)

### 1. AUTHORING FORMAT

Use plain text `.md` or `.txt` input files with structured step blocks:

```
# Slide Title
1. Step one instruction here.
2. Step two instruction here.
![optional_image](image1.png)
```

- Supports future export to:
  - PowerPoint (`.pptx`) via Python-pptx
  - PDF via Pandoc
  - HTML/Markdown for quick reviews

### 2. RAL STYLE RULES (APPLIED PER SLIDE)

- **Title:**
  - Font: Arial
  - Size: 20pt, Bold
- **Background:**
  - Light Gray container (RGB: #F9FAFB)
  - Border: Dark Teal (RGB: #00463C), 0.75pt
- **Step Indicators:**
  - Circle: Red fill (RGB: #FF3B30)
  - Text: White, Arial Bold, Centered
- **Text Block:**
  - Font: Arial
  - Size: 14pt
- **Image:**
  - Rounded corners
  - Border: 0.75pt Dark Teal

### 3. AUTOMATION STEPS

- Parse input `.md` or `.txt`
- For each slide:
  - Generate layout with containers
  - Insert steps as textboxes with red circle markers
  - Add cropped/styled images if present
- Output to `.pptx`
- Optional: Run `pandoc` to export to PDF/HTML/etc.

### 4. FUTURE INTEGRATIONS

- Integrate with 8825 `windsurf-project` for batch slide doc automation
- Pull slide content from structured JSON or Notion exports
- Attach `image_paths[]` per step to allow automated screenshot placement
