# Team Workspace

This folder contains all team collaboration files for 8825 system development.

---

## Structure

### `assignments/`
**Purpose:** Current work assignments and agent specs

**Contents:**
- Agent specifications (from planning phase)
- Implementation plans
- Success criteria
- PromptGen analyses

**When to use:**
- After planning phase complete
- Before starting implementation
- Reference during development

---

### `standups/`
**Purpose:** Daily async standup updates

**Format:** One file per day: `YYYY-MM-DD.md`

**Template:**
```markdown
# Daily Standup - [Date]

## [Team Member 1]
### Yesterday
- What you completed

### Today
- What you're working on

### Blockers
- Any issues

## [Team Member 2]
...
```

**When to use:**
- Every morning (5 min)
- End of day if significant updates

---

### `reviews/`
**Purpose:** Code review discussions and feedback

**Contents:**
- Review requests (using PR template)
- Reviewer feedback
- Change discussions
- Approval records

**When to use:**
- After implementation complete
- During review process
- For approval documentation

---

### `learnings/`
**Purpose:** Captured learnings from team work

**Contents:**
- What worked well
- What was challenging
- Process improvements
- Technical insights
- Protocol feedback

**Format:** One file per agent: `[agent-name]-learnings.md`

**When to use:**
- After completing an agent
- After retrospectives
- When discovering insights

---

## Quick Reference

### Starting New Work
1. Check `assignments/` for your spec
2. Post in `standups/` that you're starting
3. Follow implementation checklist

### Daily Updates
1. Create/update today's standup file
2. Post your update
3. Read other team members' updates

### Submitting Work
1. Create review request in `reviews/`
2. Use PR template
3. Post in `standups/` that review is ready

### After Completion
1. Capture learnings in `learnings/`
2. Update registry
3. Track protocol usage
