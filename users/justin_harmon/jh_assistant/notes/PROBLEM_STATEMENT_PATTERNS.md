## 🎯 Core Pattern: The TrustyBits Model

### Structure
```
[Target User] + [Pain Point] + [Root Cause] + [Current Gap]
```

### Example
**Problem**: Hiring managers get poor candidate matches from HR because they lack tools to translate team dynamics into actionable search criteria.

**Breakdown**:
- **Target User**: Hiring managers
- **Pain Point**: Poor candidate matches from HR
- **Root Cause**: Lack tools to translate team dynamics
- **Current Gap**: No actionable search criteria

## 📋 Problem Statement Templates

### Template 1: User-Centered Problem
```
[User Type] struggle with [specific challenge] because [underlying cause], 
resulting in [negative outcome].
```

**Example**:
> "Mid-level managers struggle with communicating effectively because they lack real-time response guidance, resulting in missed opportunities and damaged relationships."

### Template 2: Market Gap Problem
```
[Market/Industry] lacks [specific solution] that addresses [user need], 
creating an opportunity for [innovation type].
```

**Example**:
> "The hiring tools market lacks manager-focused intelligence that addresses team composition analysis, creating an opportunity for a new category: 'Hiring Manager Intelligence Tools.'"

### Template 3: Process Problem
```
[Current process] fails to [desired outcome] because [systemic issue], 
forcing users to [workaround] which leads to [consequence].
```

**Example**:
> "Current ATS systems fail to match candidates to team dynamics because they focus on keyword matching, forcing hiring managers to manually screen hundreds of resumes which leads to poor hires and wasted time."

### Template 4: Experience Problem
```
Users experience [pain point] when [context/situation] because [missing element], 
preventing them from [goal].
```

**Example**:
> "Users experience decision paralysis when responding to messages because they lack voice-consistent options, preventing them from communicating authentically and efficiently."

## 🔍 Discovery Questions (From PromptGen Protocol)

### Phase 1: Parse (Understanding)
1. What data do we have about this problem?
2. What are the measurable pain points?
3. Who is experiencing this problem?
4. What proof points exist?

### Phase 2: Select (Prioritizing)
1. Which aspects matter most to the target audience?
2. What context influences the problem severity?
3. What can we deemphasize?
4. What's the core vs. peripheral issue?

### Phase 3: Render (Articulating)
1. How do we frame this for different audiences?
2. What tone matches the problem severity?
3. What format best communicates the issue?
4. What visual elements support understanding?

### Phase 4: Validate (Testing)
1. Does this resonate with actual users?
2. Are we solving the right problem?
3. Is the problem statement measurable?
4. Can we prove this problem exists?

### Phase 5: Export (Finalizing)
1. Is this statement actionable?
2. Does it lead to clear solutions?
3. Can stakeholders understand it?
4. Is it memorable and shareable?

## 🎨 Design Sprint Problem Framework

### Challenge Question Format
```
"How might we [enable/help] [user type] to [achieve goal] 
[constraint/context]?"
```

**TrustyBits Example**:
> "How might we enable hiring managers to translate team dynamics into actionable search criteria that HR can execute?"

### Components:
1. **Enable/Help**: Action-oriented verb
2. **User Type**: Specific target persona
3. **Achieve Goal**: Desired outcome
4. **Constraint**: Real-world limitation

## 📊 Problem Statement Validation Checklist

### Must Have:
- [ ] Specific target user identified
- [ ] Measurable pain point described
- [ ] Root cause explained
- [ ] Current gap articulated
- [ ] Negative outcome stated

### Should Have:
- [ ] Quantifiable impact (time, money, satisfaction)
- [ ] Context or situation specified
- [ ] Existing workarounds mentioned
- [ ] Market opportunity implied

### Nice to Have:
- [ ] Competitive landscape referenced
- [ ] Scientific/research backing
- [ ] Multiple stakeholder perspectives
- [ ] Strategic positioning hint

## 🧠 Problem Mining Protocol

### From Existing Projects
1. **Identify Pain Point**: What frustration is expressed?
2. **Find Root Cause**: Why does this problem exist?
3. **Locate Gap**: What's missing in current solutions?
4. **Measure Impact**: What's the cost of not solving this?
5. **Define Scope**: What's in/out of the problem space?

### Example Mining Process (TrustyBits):
```
Pain Point: "HR sends poor candidate matches"
↓
Root Cause: "Hiring managers can't translate team needs"
↓
Gap: "No tools for team composition analysis"
↓
Impact: "Bad hires cost 2.5x annual salary"
↓
Scope: Team analysis input, search criteria output
```

## 💡 Problem Statement Patterns by Type

### 1. **Efficiency Problem**
**Pattern**: [Process] takes too long/costs too much because [bottleneck]

**Example**: 
> "Meeting summarization takes hours because manual note-taking misses context and requires extensive editing."

### 2. **Quality Problem**
**Pattern**: [Output] lacks [quality attribute] because [missing capability]

**Example**:
> "Candidate matches lack team fit consideration because ATS systems only match keywords, not team dynamics."

### 3. **Access Problem**
**Pattern**: [User group] can't access [resource/capability] because [barrier]

**Example**:
> "Individual contributors can't access strategic hiring insights because existing tools target HR departments only."

### 4. **Experience Problem**
**Pattern**: [User] experiences [negative emotion] when [situation] because [missing support]

**Example**:
> "Professionals experience communication anxiety when responding to messages because they lack voice-consistent response options."

### 5. **Scale Problem**
**Pattern**: [Solution] works at [small scale] but fails at [large scale] because [limitation]

**Example**:
> "Manual competitive analysis works for 1-2 competitors but fails for market-wide research because it's too time-intensive."

> "Professionals have trouble communicating quickly in high-stakes situations because they lack voice-consistent response options, resulting in missed opportunities and communication anxiety."

## 🎬 Quick Start: Generate a Problem Statement

### 5-Minute Exercise:
1. **Who** is experiencing the problem? (30 sec)
2. **What** specific challenge do they face? (1 min)
3. **Why** does this problem exist? (1 min)
4. **What** are they missing? (1 min)
5. **What** happens as a result? (1.5 min)

### Combine into statement:
```
[Who] experiences [What challenge] because [Why/missing element], 
resulting in [Impact].
```

## 📊 Problem Statement Quality Rubric

### Score 1-5 on each:
- **Specificity**: Is the user clearly defined?
- **Measurability**: Can we measure the pain?
- **Causality**: Is the root cause identified?
- **Actionability**: Does it suggest solutions?
- **Impact**: Is the consequence clear?

**Total Score**:
- 20-25: Excellent problem statement
- 15-19: Good, needs minor refinement
- 10-14: Needs significant work
- <10: Start over with more discovery
